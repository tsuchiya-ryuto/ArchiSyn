# ArchiSyn 要求仕様書

## 概要

ArchiSyn は、ソフトウェアアーキテクチャの視覚的モデリングから、ミドルウェア接続済みのスケルトンコード生成までをシームレスにつなぐ、エンジニアのためのアーキテクチャ設計ツールです。

### 目的

- コードの**実行処理部分**とミドルウェアの**インターフェース部分**を分離し、開発者が本来のアルゴリズムに集中できる環境を提供する
- ブロック線図ライクなグラフィカルモデリングにより、アーキテクチャレベルで設計・開発できる Simulink 風の環境を実現する
- スケルトンコード自動生成により、全体システム開発へのシームレスな接続と、設計〜実装間の手戻りを抑制する

---

## 要求仕様

### 1. 機能要件

#### 1.1 モデリング

| # | 要件 |
|---|------|
| F-1 | Simulink ライクなグラフィカルモデルを実現すること |
| F-2 | ノード間の接続を視覚的に設計できること |
| F-3 | カスタム型を含むインターフェース型の作成・管理ができること |
| F-4 | ノードの実行周期を設定・管理できること |

#### 1.2 コード生成

| # | 要件 |
|---|------|
| F-5 | コードの実行処理部分とインターフェース部分を分離してコード生成できること |
| F-6 | ノード単位で出力言語を選択できること |
| F-7 | ミドルウェアは ROS 2 Humble を使用するが、将来的に変更可能な設計とすること |

**オートコード対応言語**

| 言語 | サポート |
|------|---------|
| C++ | ✅ |
| Python | ✅ |
| Rust | ✅ |

---

### 2. プラットフォーム要件

| # | 要件 |
|---|------|
| P-1 | Linux / macOS / Windows のクロスプラットフォームで動作するデスクトップアプリケーションであること |
| P-2 | 各プラットフォーム向けのインストーラ・アンインストーラを用意すること |

---

### 3. ファイル管理要件

| # | 要件 |
|---|------|
| FM-1 | プロジェクト全体を 1 つのファイル（拡張子: `.arcsyn`）に保存できること |
| FM-2 | `.arcsyn` ファイルのエクスポート・インポートにより、GUI 画面の完全復元と操作再開が可能であること |
| FM-3 | `.arcsyn` はバイナリ形式ではなくテキスト形式とし、Git 等によるバージョン管理と差分抽出が容易であること |

---

## 出力イメージ

コード生成時に出力されるディレクトリ構造の例を示します。

```
generated_ws/
├── src/
│   ├── my_robot_cpp_nodes/         # C++ ノード用パッケージ
│   │   ├── CMakeLists.txt
│   │   ├── package.xml
│   │   ├── interfaces/
│   │   │   └── my_robot_cpp_nodes_interfaces.h
│   │   ├── include/my_robot_cpp_nodes/
│   │   └── src/
│   │       └── sensor_fusion.cpp   # 実装部分 ※1
│   │
│   ├── my_robot_py_nodes/          # Python ノード用パッケージ
│   │   ├── setup.py
│   │   ├── package.xml
│   │   ├── interfaces/
│   │   │   └── my_robot_py_nodes_interfaces.py
│   │   └── my_robot_py_nodes/
│   │       └── controller.py       # 実装部分 ※1
│   │
│   └── my_robot_msgs/              # カスタム型 共通パッケージ
│       ├── CMakeLists.txt
│       ├── package.xml
│       └── msg/
│           └── FusedPose.msg
│
└── launch/
    └── system.launch.py
```

> **※1 実装ファイルの生成ルール**
> ファイルが存在しない場合はスケルトンコードを生成する。すでにファイルが存在する場合は上書きせず、既存の実装を保持する。
