# ArchiSyn ユーザーマニュアル

ArchiSyn は、ノードベースのグラフィカルモデリングから、ミドルウェア接続済みの
スケルトンコード生成までを行うアーキテクチャ設計ツールです。

## 1. 画面構成

```
┌────────────────────────────────────────────────────────┐
│ メニューバー: プロジェクト名 / ミドルウェア / 新規 開く 保存 / コード生成 │
├───────────────────────────────┬────────────────────────┤
│                               │ サイドバー              │
│  キャンバス（React Flow）      │  [ノード] タブ: プロパティ │
│  ・ノード追加ボタン（左上）     │  [型] タブ: カスタム型     │
│  ・MiniMap / ズーム（右下等）   │                        │
└───────────────────────────────┴────────────────────────┘
```

## 2. モデリング

### ノード

- **追加**: キャンバス左上の「+ ノード追加」
- **移動**: ドラッグ
- **削除**: 選択して Delete / Backspace、またはインスペクタ最下部の「ノードを削除」
- **プロパティ編集**（ノードを選択 → サイドバー「ノード」タブ）:
  - **ラベル** — 生成されるノード名・クラス名の元になります（例: `SensorFusion` → `sensor_fusion`）
  - **言語** — python / cpp / rust（ノード単位で選択可能）
  - **実行周期 [ms]** — `on_update()` が呼ばれる周期
  - **入力・出力ポート** — 名前と型。テキスト入力は blur / Enter で確定
  - **パラメータ** — 名前・型（bool / int64 / float64 / string）・既定値

### 接続（エッジ）

- 出力ポート（右側のハンドル）から入力ポート（左側）へドラッグ
- **型が一致しないポートは接続できません**（ドラッグ中に無効表示、確定時はエラートースト）
- エッジは選択して Delete で削除
- トピック名は自動決定: 出力は `<ノード名>/<ポート名>`、接続された入力は接続元のトピックを購読

### カスタム型（「型」タブ）

- 「+ 型を追加」で型を作成し、フィールド（名前・型）を編集
- 定義した型はポート型・フィールド型の入力候補に現れます
- ROS 2 生成時は `<プロジェクト名>_msgs/msg/<型名>.msg` になります

## 3. プロジェクトの保存と読込

- **保存 / 名前を付けて保存**: プロジェクト全体（ノード配置・接続・型・ズーム位置まで）を
  1つの `.arcsyn` ファイル（YAML テキスト）に保存します。Git での差分管理が可能です
- **開く**: `.arcsyn` を読み込み、GUI を完全復元します
- **新規**: 現在の内容を破棄して空のプロジェクトへ（確認ダイアログあり）

サンプル: `examples/demo_robot.arcsyn`

## 4. ミドルウェアの選択

メニューバーの「ミドルウェア」で切り替えます。

| ミドルウェア  | 生成物                                                                              |
| ------------- | ----------------------------------------------------------------------------------- |
| `ros2_humble` | ROS 2 Humble の colcon ワークスペース（Python / C++ / Rust）                        |
| `mock_pubsub` | ROS 不要。全ノードを 1 プロセスの Python + インメモリバスで実行（`python3 run.py`） |

## 5. コード生成

メニューの「**コード生成**」→ 出力先ディレクトリを選択。結果（書込 / 保護スキップ / 警告）が
ダイアログで表示されます。

### 生成物の構成（ros2_humble の例）

```
出力先/
├── src/
│   ├── <project>_msgs/            # カスタム型（共通メッセージパッケージ）
│   ├── <project>_py_nodes/        # Python ノード
│   │   └── <project>_py_nodes/<ノード名>/
│   │       ├── interfaces.py      # インターフェース部（毎回再生成・編集不可）
│   │       └── <ノード名>.py       # 実装部（ここに書く・再生成でも保護）
│   ├── <project>_cpp_nodes/       # C++ ノード（src/<ノード名>/interfaces.hpp + .cpp）
│   └── <project>_rust_nodes/      # Rust ノード（src/<ノード名>/interfaces.rs + .rs）
└── launch/system.launch.py        # 全ノード起動
```

### 実装ファイル保護

- **インターフェース部**（Sub/Pub・タイマ・パラメータの配線）は生成のたびに上書きされます
- **実装部**（`<ノード名>.py` / `.cpp` / `.rs`）は初回のみ生成され、以後は上書きされません。
  設計を変更して再生成しても、書いたアルゴリズムはそのまま残ります

### アルゴリズムの書き方

各実装部で以下をオーバーライドします:

- `on_update()` — 実行周期ごとの処理。入力は `self.<入力ポート名>`（最新の受信メッセージ、
  未受信なら None/nullptr）、出力は `publish_<出力ポート名>(msg)`
- `on_<入力ポート名>(msg)` — 受信時フック（Python / C++。必要な場合のみ）
- パラメータは `self.<名前>`（Python）/ `<名前>()`（C++ / Rust）で取得

## 6. 生成物のビルドと実行

### ros2_humble（Python / C++）

```bash
docker run --rm -it -v <出力先>:/ws -w /ws osrf/ros:humble-desktop-full bash -c \
  "source /opt/ros/humble/setup.bash && colcon build && \
   source install/setup.bash && ros2 launch launch/system.launch.py"
```

### ros2_humble（Rust ノードを含む場合）

Rust ノードのビルドには ros2_rust の underlay が必要です。専用イメージを使います:

```bash
# 初回のみ（イメージ作成に 15〜20 分）
docker build -t archisyn-humble-rust -f docker/humble-rust.Dockerfile .

docker run --rm -it -v <出力先>:/ws -w /ws archisyn-humble-rust bash -c \
  "source /opt/ros2_rust_ws/install/setup.bash && colcon build && \
   source install/setup.bash && ros2 launch launch/system.launch.py"
```

### mock_pubsub

```bash
cd <出力先> && python3 run.py   # ROS 不要。Ctrl+C で終了
```

## 7. 既存システムからのインポート（逆生成）

実行中の ROS 2 システムを解析して、プロジェクトを復元できます（rqt_graph と同じ情報源）。
ArchiSyn で作っていない手書きのシステムでも、言語を問わず取り込めます。

1. 対象システムを起動した ROS 2 環境で解析スクリプトを実行:

   ```bash
   python3 tools/introspect.py -o graph.json
   # Docker 内のシステムなら: -v ./tools:/tools を付けて python3 /tools/introspect.py
   ```

2. ArchiSyn のメニュー「**グラフ取込**」から `graph.json` を読み込む
3. ノード・ポート・接続・namespace・パラメータが復元され、データフロー順に自動配置される

または、実行しなくても **Python ソースから**復元できます:
メニュー「**ソース取込**」で ROS 2 パッケージのディレクトリを選ぶと、
rclpy の create_publisher / create_subscription / create_timer / declare_parameter を
静的解析してノード・ポート・周期・パラメータ・接続を復元します
（ArchiSyn 生成物のラウンドトリップにも対応。C++/Rust ソースは将来対応）。

> 取得できないもの（インポート後に確認・修正してください）:
>
> - **言語** — 全ノード python に仮設定
> - **実行周期** — 100 ms に仮設定
> - **カスタム型のフィールド定義** — 型名（pkg/Type）の参照として取り込まれます

## 8. トラブルシューティング

| 症状                                | 対処                                                                                          |
| ----------------------------------- | --------------------------------------------------------------------------------------------- |
| ポート同士が接続できない            | 両ポートの型が完全一致しているか確認（エラートーストに型が表示されます）                      |
| 「型 X を解決できません」で生成失敗 | ポート型は `pkg/Type` 形式（例: `sensor_msgs/Imu`）か、型エディタで定義したカスタム型名にする |
| 「未対応のミドルウェアです」        | `.arcsyn` の `middleware:` がセレクタの選択肢（対応一覧はエラーに表示）にあるか確認           |
| Rust ノードで colcon build が失敗   | 標準の Humble イメージには Rust toolchain がありません。`archisyn-humble-rust` イメージを使用 |
| 実装部を作り直したい                | 対象の実装ファイルを削除してから再生成（保護は「ファイルが存在するか」で判定）                |
